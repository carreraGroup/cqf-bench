<Aside type="note" title="Early preview">
CQF Bench is a transparent benchmark harness, not a certification program. The
scenario catalog, adapter contract, and report format are usable but still
evolving. Results should be interpreted as reproducible benchmark evidence — not
certification, regulatory validation, or vendor endorsement. All included data is
synthetic and not derived from real patient, customer, or production data.
</Aside>

## Get CQF Bench

<p class="landing-lead">
  Clone the harness, stand up a reference engine with Docker, then run your first
  benchmark. No published pip package — the repo <em>is</em> the distribution.
</p>

<div class="landing-get-panel">

<Tabs syncKey="get-cqf-bench">

<TabItem label="Clone repository">

CQF Bench is Apache-2.0 open source. Clone and configure a local engines file:

```bash
git clone https://github.com/carreraGroup/cqf-bench.git
cd cqf-bench
cp bench/config/engines.example.yaml bench/config/local.engines.yaml
```

**Requirements:** Python 3.10+, Docker (for bundled engine images), Git.

</TabItem>

<TabItem label="Docker engine">

The fastest path to a running CQF server is [HAPI CQF Ruler](/cqf-bench/engines/hapi-cqf-ruler/):

```bash
python scripts/manage_engines.py bootstrap \
  --engines bench/config/local.engines.yaml \
  --engine hapi-cqf-ruler-local
```

Image: `alphora/cqf-ruler:latest` on port **8081**.

</TabItem>

<TabItem label="Run a benchmark">

```bash
scripts/bootstrap_python_env.sh --recreate
source .venv/bin/activate

python scripts/run_benchmark.py \
  --run-phase execute \
  --engines bench/config/local.engines.yaml \
  --suite bench/scenarios/tpcqf/suite.yaml \
  --scale 100 \
  --filter-engine hapi-cqf-ruler-local
```

Reports: `results/<run_id>.json` and `results/<run_id>.md`.

</TabItem>

</Tabs>

## Built for comparing engines

<p class="landing-lead">
  Born for teams evaluating CQF servers, CQL implementers, and anyone who needs
  reproducible numbers — not ad-hoc Postman collections.
</p>

<div class="landing-pill-grid">
  <div class="landing-pill">
    <strong>Reproducible</strong>
    <p>Deterministic synthetic data, suite hashing, and git commit provenance in every report.</p>
  </div>
  <div class="landing-pill">
    <strong>Conformance + performance</strong>
    <p>Untimed HTTP conformance matrices separate from timed capability runs — only PASS rows get latency.</p>
  </div>
  <div class="landing-pill">
    <strong>Engine adapters</strong>
    <p>One scenario definition runs on HAPI CQF Ruler, Mercury, or other FHIR CQF servers via adapters.</p>
  </div>
  <div class="landing-pill">
    <strong>Side-by-side reports</strong>
    <p>JSON and Markdown matrices compare every engine on the same suite and scale.</p>
  </div>
</div>
